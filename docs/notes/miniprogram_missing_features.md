# 微信小程序 (DailyMovie) 功能缺失及完善建议

已触发 `using-superpowers`，结合 `wechat-miniprogram-expert` 与 `brainstorming` 技能对当前项目进行分析。

## 当前状态分析
基于 `app.json` 的配置，小程序当前具备了核心内容浏览链路：
- **底部 Tab**：今日 (`index`)、AI推荐 (`ai`)、我的 (`profile`)
- **次级页面**：搜索 (`search`)、电影详情 (`detail`)

## 建议补充的核心模块（按优先级排序）

### 方案 1：增强“用户留存”与数据沉淀 (推荐优先 ⭐)
目前项目已跑通 TMDB 数据和基本展示，但 `profile` 页可能较为单薄。
- **缺失功能**：收藏夹 (Favorites)、想看列表 (Watchlist)、已看列表 (Watched)。
- **Trade-offs**：需要在后端（FastAPI + SQLite）新增用户相关的数据库表，并实现微信登录（`wx.login`）以绑定用户体系。开发工作量中等，但对 DAU 留存提升最明显。

### 方案 2：利用微信生态的“社交分享”能力 (见效最快 🚀)
作为小程序，尚未完全发挥微信的社交裂变优势。
- **缺失功能**：生成带有小程序码的“电影台词/推荐海报”（利用 Canvas 绘制）；定制 `onShareAppMessage` 微信转发卡片。
- **Trade-offs**：纯前端工作，无需后端修改。开发周期短，能立刻提升小程序的自然裂变带来的新用户。

### 方案 3：丰富“内容发现”路径 (体验优化 💡)
目前获取电影的路径较为单一（每日一推、AI推荐、精准搜索）。为了满足泛搜索用户：
- **缺失功能**：分类筛选功能（按类型、年份、地区等标签组合筛选）；或者“高分电影榜单”页。
- **Trade-offs**：需要与 TMDB API 进一步对接 `discover` 接口，UI 上需要设计水平滑动的标签筛选项。工作量主要在调优页面连贯性。

### 方案 4：通知与召回
- **缺失功能**：微信订阅消息推送（“每日提醒我看一部电影”）。
- **Trade-offs**：需要后端实现定时任务（可使用 Python 的 `APScheduler` 或 Celery）及微信服务端的推送接口对接。

---

> 建议我们先从 **方案1（用户留存：收藏/想看列表）** 或 **方案2（社交分享：精美分享海报）** 开始。
